## 项目交付说明书

**日期：** 2026年4月24日

## 一、项目概述

### 1.1 项目名称

**知行学堂 — 基于SpringBoot与大语言模型的智能学习助手问答平台**

### 1.2 项目背景

在高校教育数字化转型背景下，传统教学模式存在三大痛点：

| 痛点 | 具体表现 |
|------|---------|
| 答疑效率低 | 教师线下答疑时间有限，学生问题积压，难以及时响应 |
| 资源分散 | 课程资料格式多样（PDF/Word/PPT），缺乏系统化整合与检索手段 |
| 教学针对性不足 | 教师难以掌握每个学生的薄弱知识点，答疑和辅导缺乏个性化 |

### 1.3 项目目标

构建一个融合大语言模型（LLM）与检索增强生成（RAG）技术的智能学习助手平台，实现：

- **智能问答**：基于课程内容的精准AI答疑，支持全网问答和个性化问答双模式对比
- **资源管理**：课程资源的统一上传、审核、浏览和下载
- **知识点梳理**：基于课程资源自动提取和展示知识点体系
- **师生协同**：学生可发起人工答疑请求，教师在线回复，形成完整的教与学闭环
- **个性化学习**：错题本、学习计划、学习记录等个性化学习工具

### 1.4 用户角色

系统支持三种用户角色，每种角色拥有独立的界面主题和功能模块：

| 角色 | 界面主题色 | 核心定位 |
|------|-----------|---------|
| **学生** | 橙色 | 学习者——使用平台进行课程学习、智能问答、错题管理 |
| **教师** | 绿色 | 教学者——处理答疑请求、查看学生数据、发布课程重点 |
| **管理员** | 紫色 | 管理者——用户管理、资源审核、系统监控与数据统计 |

### 2.2 模块一：用户认证与权限管理

#### F-AUTH-001 用户注册

| 项目 | 说明 |
|------|------|
| **功能描述** | 支持学生、教师通过学号/工号注册账号 |
| **输入项** | 角色（学生/教师）、学号/工号、姓名、手机号、邮箱、密码、确认密码 |
| **业务规则** | ① 学号为10位数字，工号为6位数字<br>② 手机号11位，符合运营商号段<br>③ 邮箱格式正确<br>④ 密码至少6位<br>⑤ 学号/工号/手机号/邮箱全局唯一 |
| **注册流程** | 填写信息 → 表单验证 → 提交注册 → 注册成功 → 跳转登录页 |

#### F-AUTH-002 用户登录

| 项目 | 说明 |
|------|------|
| **功能描述** | 支持账号密码登录，登录后根据角色跳转到对应首页 |
| **输入项** | 用户名（学号/工号）、密码 |
| **业务规则** | ① 密码使用BCrypt加密存储<br>② 登录成功生成JWT Token，有效期24小时<br>③ Token存储在浏览器localStorage中<br>④ 每次请求自动携带Token进行身份验证 |
| **登录流程** | 输入账号密码 → 选择角色 → 点击登录 → 验证通过 → 跳转对应首页 |
| **异常处理** | 用户名或密码错误时显示提示信息 |

#### F-AUTH-003 个人信息管理

| 项目 | 说明 |
|------|------|
| **功能描述** | 查看和修改个人信息、修改密码、上传头像 |
| **功能点** | ① 查看个人基本信息（姓名、用户名、手机、邮箱）<br>② 修改手机号、邮箱<br>③ 修改密码（需验证旧密码）<br>④ 上传/更换头像 |
| **适用角色** | 学生、教师、管理员均适用（三端分别有独立的个人中心页面） |

### 2.4 模块三：智能问答与知识服务

#### F-QA-001 智能问答（核心功能）

| 项目 | 说明 |
|------|------|
| **功能描述** | 基于RAG技术的双板块智能问答，学生输入问题后同时获得两种答案 |
| **输入项** | 问题文本（10-500字） |
| **双板块设计** | **全网问答**：调用大语言模型返回通用答案<br>**个人问答**：结合个人学习记录和课程内容返回个性化答案，并识别薄弱知识点和推荐资源 |
| **输出内容** | ① AI生成的答案文本<br>② 答案来源<br>③ 置信度评分<br>④ 薄弱知识点（个人问答）<br>⑤ 推荐学习资源（个人问答）<br>⑥ 两板块答案对比 |
| **交互特性** | ① 支持Ctrl+Enter快捷发送<br>② 实时字符计数<br>③ 答案加载动画<br>④ 历史对话自动滚动 |
| **页面入口** | 学生端「智能问答」 |

#### F-QA-002 问答历史

| 项目 | 说明 |
|------|------|
| **功能描述** | 查看历史问答记录，支持搜索、筛选和删除 |
| **功能点** | ① 按关键词搜索历史问题<br>② 按类型筛选（全网问答/个人问答）<br>③ 查看问答详情<br>④ 删除历史记录<br>⑤ 分页浏览 |
| **页面入口** | 学生端「问答历史」 |

#### F-QA-003 知识点自动梳理

| 项目 | 说明 |
|------|------|
| **功能描述** | 展示课程的知识点体系 |
| **功能点** | ① 按课程查看知识点列表<br>② 显示知识点层级关系<br>③ 标注知识点重要程度 |
| **页面入口** | 学生端「课程详情」→ 知识点列表 |

#### F-QA-004 学习计划生成

| 项目 | 说明 |
|------|------|
| **功能描述** | 选择课程和天数，AI自动生成个性化学习计划 |
| **输入项** | 目标课程、计划天数（1-90天） |

| Nginx | — | 反向代理（生产环境推荐） |

### 8.2 部署步骤

```bash
# 1. 初始化数据库
mysql -u root -p < 数据库/init.sql

# 2. 构建后端
cd 后端/backend
mvn clean install -DskipTests

# 3. 启动后端（默认端口8080）
cd la-application
mvn spring-boot:run
# 或使用JAR包运行：
java -jar target/la-application-1.0.0.jar --spring.profiles.active=prod

# 4. 构建前端
cd 前端
npm install
npm run build          # 输出到 dist/ 目录

# 5. 配置Nginx反向代理
# 将前端dist/部署到Nginx
# 将/api请求代理到后端8080端口
```
