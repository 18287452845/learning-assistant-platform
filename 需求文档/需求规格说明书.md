# 基于SpringBoot与大语言模型的学习助手与问答平台 - 需求规格说明书

## 1. 项目概述

### 1.1 项目背景
在教育数字化转型背景下，高校课程教学存在三大痛点：
1. 课程答疑效率低：教师线下答疑时间有限，学生问题难以及时响应
2. 学习资源分散：课程资源格式多样，缺乏系统化整合
3. 教学针对性不足：教师难以掌握学生薄弱点，答疑缺乏针对性

### 1.2 项目目标
构建一个融合大语言模型与RAG技术的智能学习助手平台，实现：
- 课程资源的系统化管理
- 基于课程内容的精准智能问答
- 知识点的自动化梳理与个性化推荐
- 师生教与学的有效协同

### 1.3 项目范围
- 学生端：学习资源获取、智能问答、个人学习管理
- 教师端：答疑管理、教学辅助、数据统计
- 管理员端：用户管理、资源审核、系统监控

---

## 2. 用户角色定义

### 2.1 学生
**角色描述**：使用平台进行课程学习和问题咨询的高校学生

**核心需求**：
- 上传课程资料，构建个人知识库
- 获得基于课程内容的精准答疑
- 查看知识点梳理和学习计划
- 管理学习记录和错题本

### 2.2 教师
**角色描述**：负责课程答疑和教学辅导的高校教师

**核心需求**：
- 处理学生人工答疑请求
- 审核和完善智能问答结果
- 发布课程重点和解析
- 统计分析高频问题

### 2.3 管理员
**角色描述**：负责平台运维和管理的系统管理员

**核心需求**：
- 管理用户账号和权限
- 审核课程资源
- 监控系统运行状态
- 统计分析平台数据

## 3. 功能需求

### 3.1 用户认证与权限管理模块

#### 3.1.1 用户注册
| 功能编号 | F-AUTH-001 |
|---------|-----------|
| 功能名称 | 用户注册 |
| 优先级 | 高 |
| 功能描述 | 支持学生、教师注册账号 |
| 前置条件 | 无 |
| 输入 | 姓名、学号/工号、手机号、邮箱、密码、角色类型 |
| 处理流程 | 1. 验证输入信息格式<br>2. 检查学号/工号唯一性<br>3. 检查手机号、邮箱唯一性<br>4. 密码加密存储<br>5. 创建用户记录 |
| 输出 | 注册成功/失败提示 |
| 业务规则 | 1. 学号为10位数字<br>2. 工号为6位数字<br>3. 手机号11位，符合运营商号段<br>4. 邮箱格式正确<br>5. 密码至少8位，包含字母和数字 |
| 异常处理 | 1. 学号/工号已存在：提示已注册<br>2. 手机号/邮箱已存在：提示已被使用 |

#### 3.1.2 用户登录
| 功能编号 | F-AUTH-002 |
|---------|-----------|
| 功能名称 | 用户登录 |
| 优先级 | 高 |
| 功能描述 | 支持账号密码登录和人脸登录 |
| 前置条件 | 用户已注册 |
| 输入 | 方式1：账号+密码<br>方式2：人脸图像 |
| 处理流程 | **账号密码登录**：<br>1. 验证账号格式<br>2. 查询用户记录<br>3. 验证密码<br>4. 生成JWT Token<br>5. 记录登录日志<br><br>**人脸登录**：<br>1. 采集人脸图像<br>2. 调用百度云人脸识别API<br>3. 比对人脸特征<br>4. 验证通过后生成Token |
| 输出 | 登录成功：跳转首页<br>登录失败：错误提示 |
| 业务规则 | 1. 连续5次密码错误，锁定账号30分钟<br>2. Token有效期24小时<br>3. 人脸相似度阈值≥95% |

#### 3.1.3 个人信息管理
| 功能编号 | F-AUTH-003 |
|---------|-----------|
| 功能名称 | 个人信息管理 |
| 优先级 | 中 |
| 功能描述 | 用户查看和修改个人信息 |
| 功能点 | 1. 查看个人基本信息<br>2. 修改手机号、邮箱<br>3. 修改密码<br>4. 上传/修改头像<br>5. 绑定/解绑人脸 |

## 4. 非功能需求

### 4.1 性能需求
| 指标 | 要求 |
|------|------|
| 页面响应时间 | ≤2秒 |
| API响应时间 | ≤500ms（问答接口≤5秒） |
| 并发用户数 | ≥100 |
| 文件上传速度 | ≥1MB/s |

### 4.2 安全需求
- 用户密码加密存储（BCrypt）
- JWT Token认证
- HTTPS传输加密
- SQL注入防护
- XSS攻击防护
- 敏感操作日志记录

### 4.3 可用性需求
- 系统可用率 ≥ 99%
- 数据每日备份
- 异常自动告警

### 4.4 兼容性需求
- 浏览器：Chrome 90+、Edge 90+、Firefox 88+、Safari 14+
- 屏幕分辨率：≥1366×768
